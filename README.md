# University Student Support Assistant
### IS365 Practical Assignment — Full-Stack Pipeline for Deploying a Self-Hosted LLM Application

---

## Project Overview

This project is a self-hosted AI-powered University Student Support Assistant built as part of the IS365 Practical Assignment. The system allows students to ask questions about university services and receive intelligent responses generated by a locally hosted Large Language Model (LLM) running entirely on the local machine.

The main focus of this project is not to build the most intelligent chatbot, but to demonstrate a complete understanding of the full pipeline required to deploy an LLM-based application in a professional environment.

## Prerequisites

Make sure the following are installed on your machine before proceeding:

- [Python 3.10+](https://www.python.org/downloads/) — tick **Add Python to PATH** during install
- [Node.js LTS](https://nodejs.org/) — includes npm
- [Ollama](https://ollama.com/download) — for running the local LLM
- [Visual Studio Code](https://code.visualstudio.com/) — recommended code editor
- [Git](https://git-scm.com/) — for cloning the repository

## Setup Instructions

### 1. Clone the Repository

```bash
git clone https://github.com/G-Abel11/Student-Support-LLM.git
cd student-support-llm
```

### 2. Create and Activate Python Virtual Environment

```bash
python -m venv venv
venv\Scripts\activate
```

You should see `(venv)` appear at the start of your terminal prompt.

### 3. Install Python Dependencies

```bash
pip install -r requirements.txt
```

### 4. Pull the LLM Model

```bash
ollama pull llama3.2:1b
```

This downloads the model (~1 GB). Only required once.

### 5. Install Frontend Dependencies

```bash
cd frontend
npm install
cd ..
```

## Running the Application

You need **two terminals** running at the same time.

### Terminal 1 — Start the Backend

```bash
cd backend
python main.py
```

Expected output:INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Started reloader process
### Terminal 2 — Start the Frontend

```bash
cd frontend
npm run dev
```

Expected output:VITE v5.x.x  ready
➜  Local: http://localhost:5173/
Ollama runs automatically as a background service after installation — no extra terminal needed.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/` | Root welcome message |
| GET | `/health` | Backend health check |
| POST | `/ask` | Submit a question, receive an answer |

### Example Request — POST /ask

```json
{
  "question": "How do I register for courses?"
}
```

### Example Response

```json
{
  "question": "How do I register for courses?",
  "answer": "To register for courses, log in to the student portal...",
  "timestamp": "2026-01-01T12:00:00.000000"
}
```

## Error Handling

The system handles the following situations gracefully:

| Situation | Behaviour |
|-----------|-----------|
| Backend not running | Frontend shows connection error message |
| Model not running | Backend returns HTTP 503 with clear message |
| Empty question submitted | Frontend blocks submission and prompts user |
| Slow LLM response | Frontend displays loading spinner |

## Logging

All interactions are recorded to `backend/logs/app.log` with the following format:YYYY-MM-DD HH:MM:SS | LEVEL    | MESSAGE
Recorded events:
- Application startup and shutdown
- Every question received
- First 120 characters of every answer
- Rejected empty questions
- All errors with full details
